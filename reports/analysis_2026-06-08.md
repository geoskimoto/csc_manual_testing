## Test Bugs _(test needs updating, not the app)_

### `test_8_2_stripe_declined_card`
- **Description:** Test skipped with "Stripe card fields not present at checkout." Known baseline: Stripe Link injects a second iframe into `#card-number-element`, causing the test's setup to not find card fields in the expected state. The site behavior is correct.
- **Suggested fix:** Use `page.frame_locator('#card-number-element iframe').first` (or filter by title attribute) in `_fill_stripe_fields()` to avoid strict mode violations from the double iframe.

### `test_32_8_invoice_payment_page_loads_stripe`, `test_32_9_bob_pays_invoice_with_test_card`, `test_32_11_full_assign_and_pay_workflow`
- **Description:** All three fail on the same locator in `helpers.py:143`: `a[href*="/billing/invoices/"][href*="/pay/"]`. The selector resolves to a notification list item (`class="list-group-item notification-item"`) that happens to link to the invoice pay URL — this element is inside the collapsed notification dropdown and is never visible. The test finds the right URL but in the wrong DOM context. The actual invoice pay link in the main content is not being matched because the notification item takes priority.
- **Suggested fix:** Scope the locator to the main content area, e.g., `page.locator('main, .container, #main-content').locator('a[href*="/billing/invoices/"][href*="/pay/"]').first` — or exclude the notification panel: `page.locator('a[href*="/billing/invoices/"][href*="/pay/"]:not(.notification-item)')`. Inspect the invoice detail/list page DOM to confirm the correct ancestor selector.
