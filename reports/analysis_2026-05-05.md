## Test Bugs _(test needs updating, not the app)_

### `tests/test_07_09_payment.py::test_8_1_stripe_success_payment`
- **Description:** The `_fill_stripe_fields` helper uses `card_frame.locator("input").fill(card_number)`, which resolves to 6 inputs inside the Stripe iframe (hidden fields, card number, expiry month/year, CVV). Strict mode rejects ambiguous locators. The site behavior — Stripe injecting multiple inputs — is correct and expected.
- **Suggested fix:** Replace the generic `locator("input")` with a role-based selector targeting only the visible card number field: `card_frame.get_by_role("textbox", name="Credit or debit card number").fill(card_number)`. Apply the same targeted approach for expiry and CVV fields using their `data-elements-stable-field-name` attributes or `aria-label` values.

### `tests/test_07_09_payment.py::test_8_2_stripe_declined_card`
- **Description:** Same root cause as `test_8_1_stripe_success_payment` — the shared `_fill_stripe_fields` helper is called with the same ambiguous `locator("input")` that resolves to 6 elements in strict mode. Both tests fail at `test_07_09_payment.py:109`.
- **Suggested fix:** Same fix as above — update `_fill_stripe_fields` once and both tests will benefit. Use `get_by_role("textbox", name="Credit or debit card number")` for card number, and equivalent targeted selectors for expiry and CVV.
