## Real Bugs _(app needs fixing)_

### `test_15_4_billing_admin_blocked_for_booking_admin`
- **Description:** The test navigates to `/billing/admin-tool/` using a booking_admin session and expects the page to return 403 (access denied). Instead, the app returns HTTP 200 — the billing admin dashboard is accessible to booking_admin users. The test's skip guard detected this unexpected access and skipped rather than failed, but the underlying authorization gap is real: billing administration should be restricted to financial admins only, not booking admins. A booking_admin who can read or act on billing data is an overprivileged role.
- **Likely files:**
  - `billing/views.py` — check the decorator or permission mixin on the billing admin view
  - `billing/mixins.py` or `core/permissions.py` — role-based access control definitions
  - `users/models.py` — confirm `booking_admin` and `financial_admin` are distinct role groups

## Missing Data / Skips

### Stripe declined-card test — cart consumed by preceding payment test

- **Affected tests:** `tests/test_07_09_payment.py::test_8_2_stripe_declined_card`
- **Description:** `test_8_1_stripe_success_payment` ran first and completed a full payment, leaving the cart empty. When `test_8_2_stripe_declined_card` reached checkout, no Stripe card fields were rendered because there was nothing in the cart requiring payment. The test's guard condition ("Stripe card fields not present at checkout") correctly detected this state and skipped.
- **Gap:** The declined-card Stripe path is entirely untested. An independent cart pre-fill fixture (seed a booking into cart before this test runs) is needed so `test_8_2` does not depend on leftover state from `test_8_1`.

### Individual notification mark-read — unread notification consumed upstream

- **Affected tests:** `tests/test_16_notifications.py::test_16_4_individual_notification_mark_read`
- **Description:** `test_16_3_mark_all_read_action` runs before `test_16_4` and bulk-marks all notifications as read, consuming the single seeded unread notification. By the time `test_16_4` runs, no unread per-notification link is available to act on.
- **Gap:** Per-notification mark-read interaction is untested. `seed_test_data` should create at least two unread notifications so `test_16_3` and `test_16_4` each have a target.
